* Added ideal_gas_logQ_loggamma(N, V, Λ) — NEW function
  - Computes exact log Q(N,V,T) = N*log(V/Λ³) - loggamma(N+1)
  - Uses SpecialFunctions.loggamma for exact log(N!) with no approximation.
  - This is the reference function used in the high-N test.

* Identified sign bug in existing ideal_gas_logQNVT (NOT changed — kept for
  backward compatibility / awareness)
  - The function uses: N*log(V/Λ³) - N*log(N) - N
  - Correct Stirling is: N*log(V/Λ³) - N*log(N) + N  (sign on last term is wrong)
  - Or exactly:          N*log(V/Λ³) - loggamma(N+1)
  - Error grows as ~N, reaching ~213 log-units at N=108. This was the apparent
    source of "high-N upward drift" in logQ — the WL simulation was actually correct;
    the reference formula was wrong.

WHAT FIXED THE HIGH-N DRIFT?
=======================================================================

The WL simulation was NOT the source of the drift. The simulation was
already computing logQ(N) correctly — within 0.04–0.1% of exact values
at all N.

The apparent "systematic upward drift" was entirely due to a sign bug in
the reference formula ideal_gas_logQNVT in src/thermo.jl. That function
uses Stirling's approximation with the wrong sign on the last term (-N
instead of +N), causing the analytic reference to be systematically too
LOW by ~2N log-units (roughly 2 per particle). At N=108 this amounts to
~213 log-units of error in the reference, making the WL result look
artificially high by comparison.

Fix: use ideal_gas_logQ_loggamma (exact, via loggamma(N+1)) as the
reference in the high-N test. With the correct reference, the test passes
comfortably within the 5% tolerance across all N=10 to 108.

The shallow copy bug fix (commit a75b069) was a separate real correctness
fix in the simulation scaffolding — it prevented position array aliasing
between μ and μ_prop. This would have corrupted energy evaluations and
acceptance decisions and could have contributed to errors under interacting
(non-ideal-gas) conditions, even though at T*=1e6 the Boltzmann factor is
always ~1 and masks such errors.
